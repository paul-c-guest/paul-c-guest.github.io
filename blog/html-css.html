On Assigning Class And Identity

thoughts and best practices on how to apply each in your html and css

22 June 2023

Introduction

HTML looks a lot better with a bit of style. The rules that make up a style can be put together in reusable categories. By creating and applying these categories the developer can both save time, and present a consistent experience for the user.

However, when deciding to create one of these stylistic categories, there is already a choice. Should the developer create a class, or an ID?

A lifetime ago, when I was an arts degree undergraduate, we were bombarded with the concepts of class and identity. You might think an arts degree and styling your HTML with CSS would have nothing in common — but I reckon there's a good reason why the terminology of class and ID have shown up here.

Class

Arts degree academics love to talk about class! Let's use a simplified version of Karl Marx's theory of class structure in an industrialised society. Whoever you are in that society, you'll probably get pigeon-holed into one of two classes (remember, just keeping it simple here). In this structure, it's pretty likely that you're either:

  1. a worker, or
  2. a member of the elite

The bit to keep in your mind here is that many individuals can be assigned the same class.

Identity

Arts degree academics also love to talk about identity! Every individual has their own unique mix of qualifications, personality quirks, and food allergies that make them who they are. On top of that, even if it were possible to make two people exactly the same — there would now be two of them, physically forced to take different paths.

In the HTML realm, identity is shortened to 'id'. The bit to keep in your mind here is that id should only be assigned to a single individual.

Priority

Academics love to debate whether class or identity is more important! In the HTML world, it's been decided for us to avoid messy class struggles.

The most important thing to HTML is identity, and then class. Think of those animated films where the main characters are always fiercely individual, because of the unique combination of their personal traits.

In Practice

When the web developer is constructing their style categories, they can create a structure that allows for uniformity and individualism in the same room.

Uniformity, the repetition of styles on a page by the use of 'class', is an important tool to aid the website visitor. Consistency will help them quickly recognise the purpose of similar elements, like how these paragraphs are visually different from the headings.

Individualism, in this case the use of 'id', lets the developer elevate chosen elements out of mediocrity. Now freed from their dull class restrictions, they can be called up by their unique name, and wear that incredible one-of-a-kind shirt.

Proceed with Care

Many classes can be assigned to a single element! This is fine though, in fact it's normal. The element will simply pay attention to the last thing it was told to do.

Unfortunately, in the same manner, many ids can also be given to an element. This is highly unrecommended, and now it's hopefully a bit clearer why. When we call up that id, hoping it's a unique element on our page, which one would we get..?
